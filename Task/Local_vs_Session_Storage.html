Q1. Theory (2 Marks each)

1. Define localStorage and sessionStorage.

localStorage: Data is stored permanently in the browser until it is manually cleared (by code or user).
sessionStorage: Data is stored only for the session (as long as the browser tab is open).

2. Two Key Differences Between localStorage and sessionStorage:

Feature localStorage sessionStorage
Lifetime Permanent (until cleared) Temporary (tab session)
Scope All tabs & windows Current tab only

3. Why is Web Storage Considered Better Than Cookies?

More Storage Capacity: Web Storage offers much more space (typically 5-10MB) compared to cookies (~4KB).
No Automatic Server Transmission: Data in Web Storage is not sent with every HTTP request, unlike cookies, which improves performance and reduces unnecessary network traffic.

4. Can sessionStorage Share Data Across Multiple Browser Tabs? Explain.

No, sessionStorage cannot share data across tabs

It is tab-specific, meaning each tab has its own separate sessionStorage. Opening the same website in a new tab will create a new, empty sessionStorage.

5. What is the Typical Maximum Storage Capacity of localStorage and sessionStorage?

Both localStorage and sessionStorage typically offer about 5MB per origin (domain).
This limit may vary slightly by browser, but 5MB is a common default.
